用户接口文档

获取个人信息

用户进入指定“我的界面”时，自动获取当前用户的信息，包括用户昵称、头像该文章的内容（作者信息、文章信息、文章标题与正文），并显示在界面中。

前端请求

  • 请求地址：
  • 请求方式：POST
  • 请求参数：
jchen: '879646529' // 用户账号

后台响应

  • 对应视图：view_user
  • 响应内容：
{
    "code": "1",
    "msg": "",
    "data": {
        "nickname": "游学者、墨槿",
        "head": "kaiyan/head/879646528.jpg",
        "brief": "人生不是戏剧，而我亦非主角",
        "countFocuus": 82,
        "countFans": 2365,
        "countLike": 15,
        "countArticle": 2,
        "countSubject": 1,
        "indexback": "kaiyan/back/879646529.jpg",
        "countWords": 8452,
        "countLiked": 4521
    }
}

// 失败响应
{
    "code": "0",
    "msg": "具体原因",
    "data": []
}

修改用户头像

用户进入设置界面并点击个人头像，即可上传照片以修改自身用户头像。

前端请求

  • 请求地址：
  • 请求方式：POST
  • 请求参数：
hcnsg: '879646529'  // 用户账号
yshuc: ...          // base64 图片数据

后台响应

  • 对应数据表：ruser
  • 响应内容：
// 成功响应
{
    "code": "1",
    "msg": "修改头像成功！",
    "data": []
}

// 失败响应
{
    "code": "0",
    "msg": "具体原因",
    "data": []
}

【说明】：后台在收到前端传送的 base64 图片数据后写入到指定的文件夹（kaiyan/back）中，并以用户账号.jpg 的方式进行命名。将路径保存到 ruser 数据表中。

修改登录密码

操作过程同修改用户头像。

前端请求

  • 请求地址：
  • 请求方式：POST
  • 请求参数：
jksnd: '879646529'        // 用户账号
nklsh: '加密后的密码'     // 原密码
wrqsd: '加密后的密码'     // 新密码

后台响应

  • 对应数据表：ruser
  • 响应内容：
// 成功响应
{
    "code": "1",
    "msg": "修改登录密码成功！",
    "data": []
}

// 失败响应
{
    "code": "0",
    "msg": "具体原因",
    "data": []
}

【过程】：后台收到前端发送的用户账号和旧密码，先进行比对。比对正确后再将密码更改为新密码。

获取动态信息

个人主页的动态栏目记录了当前用户最近的一些行为，例如喜欢了某某文章、关注了某某作者等。每次获取最近的十条记录。

前端请求

  • 请求地址：
  • 请求方式：POST
  • 请求参数：
nhcjs: '879646528' // 用户账号
asxcd: '2018-03-05 18:18' // 已获取动态中的最早日期

  • 响应内容：

            "key": "2",
            "value": "Web 前端之路",
            "type": 3,
            "date": "2018-03-08 18:18"
        }, {
            "key": "8796465291803254565",
            "value": "《浅谈HTML5 Geolocation》",
            "type": 2,
            "date": "2018-03-07 18:18"
